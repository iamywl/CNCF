Mermaid.js — 코드로 그리는 다이어그램

아래의 모든 다이어그램은 텍스트 코드에서 자동 생성되었습니다.

1. 플로우차트 (Flowchart)

프로세스 흐름, 의사결정 트리, 알고리즘을 표현할 때 사용합니다.
graph TD Start([시작]) --> InputAge[/나이 입력/] InputAge --> CheckAge{19세 이상?} CheckAge -->|Yes| Adult[성인 인증 완료] CheckAge -->|No| Minor[미성년자 안내] Adult --> Purchase[구매 진행] Minor --> End([종료])
graph TD
    Start([시작]) --> InputAge[/나이 입력/]
    InputAge --> CheckAge{19세 이상?}
    CheckAge -->|Yes| Adult[성인 인증 완료]
    CheckAge -->|No| Minor[미성년자 안내]
    Adult --> Purchase[구매 진행]
    Minor --> End([종료])

2. 시퀀스 다이어그램

객체 간 메시지 교환 순서를 시간 흐름에 따라 표현합니다.
sequenceDiagram participant B as Browser participant S as Server participant D as Database B->>S: GET /api/users activate S S->>D: SELECT * FROM users D-->>S: 결과 반환 S-->>B: JSON 응답 deactivate S
sequenceDiagram
    participant B as Browser
    participant S as Server
    participant D as Database
    B->>S: GET /api/users
    activate S
    S->>D: SELECT * FROM users
    D-->>S: 결과 반환
    S-->>B: JSON 응답
    deactivate S

3. 클래스 다이어그램

OOP의 클래스 구조, 상속, 연관 관계를 표현합니다.
classDiagram
    class Animal {
        +String name
        +int age
        +makeSound() void
    }
    class Dog {
        +String breed
        +fetch() void
        +makeSound() void
    }
    class Cat {
        +bool isIndoor
        +purr() void
        +makeSound() void
    }
    class Owner {
        +String name
        +List~Animal~ pets
        +adopt(Animal) void
    }

    Animal <|-- Dog : 상속
    Animal <|-- Cat : 상속
    Owner "1" --> "*" Animal : 소유

4. 상태 다이어그램

주문, 결제 등 상태 전이를 표현할 때 유용합니다.
stateDiagram-v2
    [*] --> 주문접수
    주문접수 --> 결제대기 : 주문 확인
    결제대기 --> 결제완료 : 결제 성공
    결제대기 --> 주문취소 : 결제 실패
    결제완료 --> 배송준비 : 상품 포장
    배송준비 --> 배송중 : 택배 접수
    배송중 --> 배송완료 : 수령 확인
    배송완료 --> [*]
    주문취소 --> [*]

    배송중 --> 반품요청 : 반품 신청
    반품요청 --> 반품완료 : 반품 처리
    반품완료 --> [*]

5. ERD (Entity Relationship Diagram)

데이터베이스 테이블 간 관계를 표현합니다.
erDiagram
    CUSTOMER {
        int id PK
        string name
        string email
    }
    ORDER {
        int id PK
        int customer_id FK
        date created_at
    }
    PRODUCT {
        int id PK
        string name
        float price
    }
    ORDER_ITEM {
        int order_id FK
        int product_id FK
        int quantity
    }

    CUSTOMER ||--o{ ORDER : "주문"
    ORDER ||--|{ ORDER_ITEM : "포함"
    PRODUCT ||--o{ ORDER_ITEM : "주문됨"

6. 간트 차트

프로젝트 일정 관리에 사용합니다.
gantt

    dateFormat  YYYY-MM-DD
    section 기획
        요구사항 분석     :a1, 2024-01-01, 7d
        화면 설계         :a2, after a1, 5d
    section 개발
        백엔드 API       :b1, after a2, 14d
        프론트엔드 UI    :b2, after a2, 14d
        DB 설계          :b3, after a2, 7d
    section 테스트
        통합 테스트       :c1, after b1, 7d
        UAT              :c2, after c1, 5d
    section 배포
        스테이징 배포     :d1, after c2, 2d
        프로덕션 배포     :d2, after d1, 1d
