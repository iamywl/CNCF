# 14. Runbook — 운영 문서 (배포, 환경변수, 트러블슈팅)

## 개념

Runbook은 시스템 **운영에 필요한 절차를 문서화**한 것입니다.
배포 프로세스, 환경 변수 설정, 장애 대응(트러블슈팅)을
**누구나 따라할 수 있는 단계별 절차**로 기록합니다.

## 왜 필요한가?

- **속인화 방지**: "그 사람만 아는" 절차를 없앰
- **장애 대응 시간 단축**: 새벽 3시에 장애가 나도 Runbook을 보고 대응 가능
- **온보딩 가속**: 신규 입사자가 스스로 환경 설정 가능
- **일관성 보장**: 매번 동일한 절차로 배포하여 실수 방지

## Runbook 핵심 구성

| 문서 | 내용 | 갱신 주기 |
|------|------|----------|
| 배포 가이드 | 빌드 → 테스트 → 배포 → 검증 절차 | 배포 방식 변경 시 |
| 환경 변수 가이드 | 모든 환경 변수의 용도, 기본값, 민감도 | 변수 추가/변경 시 |
| 트러블슈팅 가이드 | 증상 → 원인 → 해결 매핑 | 장애 복기 후 |
| 롤백 절차 | 배포 실패 시 이전 버전 복구 방법 | 배포 방식 변경 시 |

## POC 프로젝트

실제 프로젝트에 바로 적용할 수 있는 운영 문서 템플릿을 제공합니다.

### 파일 구조

```
14-runbook/

├── environment-variables.md    ← 환경 변수 설정 가이드

```

### 사용 방법

```bash
# 각 문서를 프로젝트의 docs/ 디렉토리에 복사하여 커스터마이징
cp 14-runbook/*.md /your-project/docs/runbook/
```

## 학습 포인트

1. **단계별 절차**: 모든 작업을 번호가 매겨진 단계로 작성 (절대 "알아서 해" 금지)
2. **복사-붙여넣기 가능**: 명령어는 바로 실행할 수 있는 형태로 작성
3. **검증 단계 필수**: 각 작업 후 "성공 확인 방법"을 명시
4. **장애 복기 → 문서 업데이트**: 매 장애 후 Runbook을 보강하는 사이클
