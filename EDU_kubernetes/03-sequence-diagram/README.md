# 03. 시퀀스 다이어그램 (Sequence Diagram)

## 개념

시퀀스 다이어그램은 **시간의 흐름에 따라** 객체(또는 시스템) 간의 **메시지 교환**을 보여줍니다.
특정 기능이 실행될 때 어떤 순서로 누가 누구에게 요청하는지를 표현합니다.

## 왜 필요한가?

- 복잡한 비즈니스 로직의 흐름을 시각적으로 이해
- API 호출 순서와 에러 처리 흐름 설계
- 코드 리뷰 시 "이 기능이 어떻게 동작하는지" 설명하는 최적의 도구

## 핵심 구성 요소

| 요소 | 설명 | Mermaid 문법 |
|------|------|-------------|
| 참여자 (Participant) | 메시지를 주고받는 객체 | `participant A` |
| 동기 메시지 | 응답을 기다리는 호출 | `A->>B: 요청` |
| 비동기 메시지 | 응답을 기다리지 않음 | `A-)B: 이벤트` |
| 응답 | 점선 화살표 | `B-->>A: 응답` |
| 활성 구간 | 처리 중인 시간 | `activate B` / `deactivate B` |
| 대안 분기 | if/else 조건 | `alt` / `else` / `end` |
| 루프 | 반복 | `loop 조건` ... `end` |
| 노트 | 보충 설명 | `Note over A,B: 설명` |

## POC 프로젝트

실제 웹 서비스에서 자주 발생하는 시나리오를 시퀀스 다이어그램으로 표현합니다.

### 실행 방법

```bash
open index.html
```

### 파일 구조

```
03-sequence-diagram/
├── README.md      ← 지금 보고 있는 파일
└── index.html     ← 4가지 시나리오 다이어그램
```

### 포함된 시나리오

1. **사용자 로그인** — 인증 흐름 (JWT 토큰 발급)
2. **상품 주문** — 결제 → 재고 차감 → 알림 흐름
3. **에러 처리** — 결제 실패 시 롤백 시나리오
4. **비동기 처리** — 이메일 발송을 큐에 위임

## 학습 포인트

1. **동기 vs 비동기**: 실선(`->>`)은 동기, 점선(`-)`)은 비동기
2. **활성 구간**: `activate`/`deactivate`로 처리 시간을 명시
3. **분기 표현**: `alt`/`else`로 조건 분기를 다이어그램에 반영
