# 01. 시스템 아키텍처 다이어그램

## 개념

시스템 아키텍처 다이어그램은 소프트웨어 시스템의 **전체 구조**를 시각적으로 표현합니다.
서버, 데이터베이스, 외부 API, 메시지 큐 등 구성 요소들이 어떻게 연결되는지를 한눈에 보여줍니다.

## 왜 필요한가?

- 새로운 팀원이 시스템 전체를 빠르게 파악할 수 있음
- 장애 발생 시 영향 범위를 신속히 판단 가능
- 설계 리뷰 및 기술 의사결정의 근거 자료

## 주요 표현 방식

### 1. C4 모델 (Context, Container, Component, Code)
가장 널리 사용되는 아키텍처 시각화 모델입니다.

```
Level 1: System Context  → 시스템과 외부 사용자/시스템 관계
Level 2: Container        → 애플리케이션, DB, 메시지 브로커 등
Level 3: Component        → 컨테이너 내부의 주요 모듈
Level 4: Code             → 클래스/함수 수준 (필요시)
```

### 2. 박스-화살표 다이어그램
가장 직관적인 형태로, 구성 요소를 박스로, 통신을 화살표로 표현합니다.

## POC 프로젝트

이 POC는 Mermaid.js를 사용하여 전형적인 웹 서비스의 아키텍처를 표현합니다.

### 실행 방법

```bash
# HTML 파일을 브라우저에서 열기
open index.html
```

### 파일 구조

```
01-architecture-diagram/
├── README.md          ← 지금 보고 있는 파일
├── index.html         ← 브라우저에서 렌더링되는 다이어그램
└── diagrams/
    ├── c4-context.md  ← C4 Level 1: 시스템 컨텍스트
    ├── c4-container.md← C4 Level 2: 컨테이너
    └── microservice.md← 마이크로서비스 아키텍처 예시
```

## 학습 포인트

1. **추상화 수준을 결정하라**: 청중에 따라 Level 1(경영진)과 Level 3(개발자)을 구분
2. **통신 방식을 명시하라**: HTTP, gRPC, AMQP 등 프로토콜을 화살표에 표기
3. **경계를 그려라**: 네트워크 경계, 보안 영역을 점선으로 구분
