3. 마이크로서비스 아키텍처 예시

목적: 대규모 시스템의 서비스 분리 전략을 보여줍니다.
핵심: 각 서비스는 독립적 배포 가능, 자체 DB 소유 (Database per Service 패턴) 
graph TB
    Client["👤 Client"]
    Gateway["🚪 API Gateway
(Kong / Nginx)"]

    subgraph "Service Mesh (Kubernetes)"
        UserSvc["👤 User Service
(Go)"]
        ProductSvc["📦 Product Service
(Node.js)"]
        OrderSvc["🛒 Order Service
(Java)"]
        PaymentSvc["💳 Payment Service
(Python)"]
        NotifySvc["🔔 Notification Service
(Node.js)"]
    end

    subgraph "Data Stores"
        UserDB[("Users DB
PostgreSQL")]
        ProductDB[("Products DB
MongoDB")]
        OrderDB[("Orders DB
PostgreSQL")]
        MQ["📨 Message Queue
(RabbitMQ)"]
    end

    Client -->|"HTTPS"| Gateway
    Gateway -->|"/api/users"| UserSvc
    Gateway -->|"/api/products"| ProductSvc
    Gateway -->|"/api/orders"| OrderSvc

    UserSvc --> UserDB
    ProductSvc --> ProductDB
    OrderSvc --> OrderDB

    OrderSvc -->|"주문 생성 이벤트"| MQ
    MQ -->|"결제 처리"| PaymentSvc
    MQ -->|"알림 전송"| NotifySvc

    style MQ fill:#ff6600,color:white
