# Streaming & Messaging

분산 시스템 간 비동기 메시지 전달, 이벤트 스트리밍, 실시간 데이터 처리 도구들.

---

## 핵심 개념

- **메시징(Messaging)**: 서비스 간 메시지를 비동기로 주고받는 것. 보내는 쪽과 받는 쪽이 동시에 살아 있을 필요 없음
- **스트리밍(Streaming)**: 연속적인 데이터 흐름을 실시간으로 처리. 메시지를 저장하고 재처리 가능
- **Pub/Sub**: Publisher가 메시지를 발행하면, 구독(Subscribe)한 Consumer가 수신하는 패턴

---

## Graduated

### CloudEvents ★5.7k
- **역할**: 이벤트 메타데이터 표준 스펙
- **왜 쓰나**: 서로 다른 시스템이 이벤트를 교환할 때 형식을 통일. "누가, 언제, 어떤 타입의 이벤트를 보냈는가"를 표준화
- **비유**: 편지 봉투의 표준 양식 — 보내는 사람, 받는 사람, 날짜 위치를 통일

---

## Incubating

### NATS ★19.2k
- **역할**: 초경량 고성능 메시징 시스템
- **특징**: 단일 바이너리, 설정 최소, 멀티테넌트, Pub/Sub + Request/Reply + Streaming(JetStream)
- **왜 쓰나**: Kafka보다 훨씬 가볍고 빠름. 설치와 운영이 극도로 간단
- **비유**: 경량 우편 시스템 — 편지(메시지)를 빠르고 가볍게 전달

### Strimzi ★5.7k
- **역할**: Kubernetes 위에서 Apache Kafka를 운영하는 Operator
- **왜 쓰나**: Kafka 클러스터 배포, 스케일링, 업그레이드, 모니터링을 K8s CRD로 선언적으로 관리
- **비유**: Kafka를 K8s 시민으로 만들어주는 번역가

### Kafka vs NATS vs RabbitMQ 비교

| | Kafka | NATS | RabbitMQ |
|---|---|---|---|
| **설계 철학** | 내구성 있는 로그 | 초경량/초고속 | 유연한 라우팅 |
| **메시지 저장** | 디스크 (영구) | 메모리 (JetStream: 디스크) | 메모리/디스크 |
| **처리량** | 매우 높음 | 매우 높음 | 높음 |
| **운영 복잡도** | 높음 (ZooKeeper/KRaft) | 매우 낮음 | 중간 |
| **적합한 용도** | 이벤트 소싱, 로그 수집, 대규모 스트리밍 | 마이크로서비스 통신, IoT, 경량 메시징 | 작업 큐, 복잡한 라우팅, 전통적 메시징 |
| **프로토콜** | 자체 프로토콜 | 자체 프로토콜 | AMQP |
