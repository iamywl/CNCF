> - 각 WU는 **독립적**이므로 최대 병렬 실행 가능

## 4. 실행 전략

### 4.1 병렬 실행 구성

```
시간 →
────────────────────────────────────────────────

Wave 1 (최대 6세션 병렬):
  Session A: WU-01 (kafka 완성)
  Session B: WU-02 (alertmanager 보강)
  Session C: WU-03 (argo-cd PoC 수정)
  Session D: WU-04 (containerd + jenkins)
  Session E: WU-05 (cilium + hubble)
  Session F: WU-06 (helm + istio)

Wave 2 (최대 5세션 병렬, Wave 1과 독립적이므로 동시 시작 가능):
  Session G: WU-07 (kubernetes 검증)
  Session H: WU-08 (grpc-go 보강)
  Session I: WU-09 (grafana + loki 검증)
  Session J: WU-10 (jaeger + tart 검증)
  Session K: WU-11 (terraform 검증)

Wave 3 (순차, Wave 1+2 완료 후):
  Session L: WU-12 (통합 검증)
```

### 4.2 최대 병렬도

- **이론적 최대**: 11세션 동시 (WU-01 ~ WU-11 모두 독립적)
- **권장**: 5~6세션 동시 (Claude Max 요금제 동시 세션 제한 고려)
- **최소**: 3세션씩 4라운드 (보수적)

### 4.3 세션별 프롬프트 템플릿

각 세션 시작 시 아래 형식으로 지시:

```
/edu-create 또는 수동 지시:

"WU-{번호} 작업을 수행합니다.
대상: {프로젝트명}_EDU
작업 유형: {완성|보강|수정|검증}
상세 내용:
1. {작업1}
2. {작업2}
3. {작업3}
완료 후: doc/edu_verification_plan/WU-{번호}_RESULT.md에 결과 기록"
```

### 4.4 결과 추적

각 WU 완료 시 결과 파일 생성:

```
doc/edu_verification_plan/
├── VERIFICATION_PLAN.md          (이 파일)

├── WU-02_RESULT.md               (alertmanager 보강 결과)
├── ...
├── WU-12_RESULT.md               (통합 검증 결과)

```
