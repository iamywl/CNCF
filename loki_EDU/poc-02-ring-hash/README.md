# PoC 02: Ring 해시 (Consistent Hash Ring)

## 개요

Loki의 dskit/ring 패키지가 구현하는 Consistent Hash Ring을 시뮬레이션한다.
Ingester 인스턴스 간에 로그 스트림을 균등하게 분산하고, 노드 추가/제거 시
최소한의 재분배만 발생하도록 보장하는 핵심 메커니즘이다.

## 시뮬레이션하는 Loki 컴포넌트

| 컴포넌트 | Loki 실제 위치 | 설명 |
|----------|---------------|------|
| Ring | `dskit/ring/ring.go` | Consistent Hash Ring 자료구조 |
| Token | `dskit/ring/model.go` | 링 위의 가상 노드 |
| Get() | `dskit/ring/ring.go` | 키 → 노드 라우팅 |
| ReplicationSet | `dskit/ring/ring.go` | 복제 인자에 따른 다중 노드 선택 |

## 핵심 알고리즘

### Consistent Hashing

```
해시 공간: 0 ─────────────────────── 2^32-1
                    (원형)

       토큰 A1    토큰 B1    토큰 C1
         │          │          │
   ──────●──────────●──────────●──────
         ↑                     ↑
      key X → Node A        key Y → Node C
   (시계 방향 첫 번째 토큰)
```

### 가상 노드(토큰)의 역할

- 물리 노드 1개 = 가상 노드 128개 (Loki 기본값)
- 가상 노드가 많을수록 분산이 균등해짐
- 표준편차가 가상 노드 수에 반비례하여 감소

## 시나리오

1. **기본 링 구성**: 3노드, 128 가상노드, 복제인자 3
2. **스트림 라우팅**: 테넌트별 스트림을 담당 노드에 배정
3. **노드 추가**: 새 노드 추가 시 ~1/N 키만 재배치
4. **노드 제거**: 노드 제거 시 해당 키가 다른 노드로 이동
5. **가상 노드 수 비교**: 1~512 가상노드에 따른 분산 균등도

## 실행 방법

```bash
go run main.go
```

## 학습 포인트

- Consistent Hashing이 단순 모듈러 해싱보다 유리한 이유
- 가상 노드(토큰)가 분산 균등도에 미치는 영향
- 이진 검색으로 O(log N) 시간에 담당 노드 탐색
- 복제 인자(Replication Factor)와 데이터 내구성의 관계
- 노드 추가/제거 시 최소 재분배 원리
