# Claude Max 최적화 골든 프롬프트 (CNCF EDU용)

이 프롬프트는 Claude Max의 **긴 컨텍스트(200k)**와 **강력한 추론 능력**을 100% 활용하면서, 불필요한 토큰 낭비를 줄이고 **500줄 이상의 Deep-dive** 출력을 유도하도록 설계되었습니다.

---

## 1. 프로젝트 초기 분석 및 아키텍처 수립용 (초기 대화)

> **[Context Package: Gemini CLI가 생성한 요약본 첨부]**
> 
> "너는 CNCF 프로젝트를 전문적으로 분석하는 시니어 클라우드 아키텍트야. 내가 제공한 [Context Package]를 바탕으로 이 프로젝트의 핵심 동작 원리를 분석하려고 해.
> 
> **작업 지침:**
> 1. 제공된 핵심 구조체(Struct)와 함수 간의 관계를 바탕으로 **전체 컴포넌트 아키텍처**를 정의해줘.
> 2. 특히 'Data Flow(데이터 흐름)'가 어떻게 시작되고 끝나는지, 핵심 루프(Main Loop)의 위치를 코드 레벨에서 짚어줘.
> 3. 분석 결과는 나중에 작성할 '심화 Deep-dive 문서(12개)'의 기초가 될 거야. 누락된 핵심 서브시스템이 없는지 확인해줘.
> 4. 답변은 한국어로, 핵심 용어는 영어와 병기해줘."

---

## 2. 500줄 이상의 심화 문서 작성용 (본 작업)

> "이제 **[특정 서브시스템명]**에 대한 심화 분석 문서(Deep-dive)를 작성할 거야. 이 문서는 교육용 자료로 사용될 예정이며, 최소 500줄 이상의 풍부한 내용을 담고 있어야 해.
> 
> **작성 규칙:**
> 1. **Why & How:** 단순히 코드를 나열하지 말고, 왜 이 설계를 선택했는지(Design Trade-off)와 실제 구현 방식(Mechanism)을 깊게 파고들어줘.
> 2. **Code Reference:** 제공된 컨텍스트에 있는 실제 파일 경로와 함수명을 인용해. (추측 금지)
> 3. **Structure:** 
>    - 도입부: 서브시스템의 역할과 필요성
>    - 핵심 데이터 구조: 주요 Struct와 필드 분석
>    - 제어 흐름: ASCII 시퀀스 다이어그램 또는 Mermaid 활용
>    - 심화 분석: 성능 최적화, 동시성 처리, 에러 핸들링 전략
> 4. **Incremental Output:** 내용이 너무 길어 끊길 것 같으면, 우선 전반부(목차~핵심 구조)를 먼저 출력하고 내 신호를 기다려줘.
> 
> 지금 시작해줘."

## 3. PoC 코드 생성 및 검증용

> "이 프로젝트의 **[특정 메커니즘]**을 시뮬레이션하는 PoC 코드를 작성해줘.
> 
> **제약 사항:**
> 1. **No External Deps:** Go 표준 라이브러리(`net/http`, `sync`, `context` 등)만 사용해.
> 2. **Minimalism:** 실제 소스코드의 복잡한 추상화는 걷어내고, 핵심 알고리즘/로직만 직관적으로 재현해.
> 3. **Executable:** `go run main.go`로 즉시 실행 가능해야 하며, 동작 과정을 `fmt.Printf`로 상세히 출력해.
> 4. **Explanation:** 코드 주석에 실제 오픈소스 소스코드의 어느 부분(파일/라인)을 모사한 것인지 명시해줘."
