# CNCF 오픈소스 정복을 위한 실전 실행 계획서 v2 (Execution Plan)

본 문서는 `마스터 플랜 v2`를 기반으로, 향후 4~6주간 진행될 분석 및 검증 작업의 세부 일정과 실행 지침을 정의한다.

## 1. 핵심 목표 및 마일스톤 (Milestones)

*   **M1 (1주차):** **Istio v2 골드 스탠다드 수립.** (v2 프레임워크의 모든 단계를 적용한 최초의 완벽한 샘플 완성)
*   **M2 (2~3주차):** **기존 11개 프로젝트 소급 검증 및 업그레이드.** (K8s, Cilium 등 핵심 프로젝트 우선순위 적용)
*   **M3 (4~5주차):** **잔여 4개 프로젝트(Kafka 등) 신규 분석 완료.**
*   **M4 (6주차):** **전체 프로젝트 최종 통합 빌드 및 기술 자산화.**

## 2. 프로젝트별 상세 실행 우선순위

| 우선순위 | 프로젝트 | 난이도 | 예상 기간 | 핵심 작업 내용 |
| :--- | :--- | :---: | :---: | :--- |
| **P0 (최우선)** | **Istio** | 최상 | 5~7일 | v2 프레임워크 전 단계 최초 적용 및 모범사례 구축 |
| **P1 (핵심)** | **Kubernetes** | 최상 | 4~5일 | PoC 커버리지 200% 확충, 스케줄러/컨트롤러 정밀 검증 |
| **P1 (핵심)** | **Cilium** | 상 | 3~4일 | eBPF 커널 로직과 PoC 간의 정합성 집중 분석 |
| **P2 (보강)** | **Argo CD / Helm** | 중 | 각 2일 | GitOps 흐름 및 템플릿 엔진 심화 분석 추가 |
| **P3 (신규)** | **Kafka** | 최상 | 5~6일 | Java/Scala 아키텍처 분석 및 분산 메시징 PoC 구현 |
| **P4 (마무리)** | **Grafana / Loki** | 중 | 각 2일 | 시각화 및 로그 엔진 서브시스템 분석 |

## 3. 주간 단위 상세 루틴 (Weekly Routine)

매주 하나의 메인 프로젝트를 완파하는 것을 원칙으로 한다.

*   **월요일 (Research Day):** Gemini Pro를 활용한 1M+ 토큰 코드 리서치. 진입점, 역사적 맥락(PR/Issue), 핵심 데이터 모델 확정.
*   **화요일 (Test Dissection):** 테스트 코드 분석을 통한 런타임 의도 파악. 클로드용 [Context Package] 생성 및 전달.
*   **수요일/목요일 (Content Generation):** Claude Max를 활용한 500줄 이상의 심화 문서(12개 이상) 및 무제한 PoC 코드 생성.
*   **금요일 (Technical Audit):** Gemini CLI를 통한 경로/심볼 정합성 전수 조사 및 PoC 실행 검증. 오류 수정.
*   **토요일 (Final Review):** 전체 문서의 기술적 완결성 최종 검토 및 `Validated` 상태 업데이트.

## 4. 도구 활용 및 토큰 관리 전략

*   **Gemini CLI (The Auditor):** 코드 검색, 파일 경로 확인, PoC 실행 등 **'물리적 검증'**에 집중. (토큰 절약을 위해 `read_file` 범위 지정 필수)
*   **Claude Max (The Writer):** Gemini가 요약해준 컨텍스트를 바탕으로 **'고품질 문서 작성'**에 집중. (메시지 제한을 고려하여 프로젝트당 3~4개 세션으로 분할 운영)
*   **Context Management:** 각 단계 완료 시 `/compact` 명령어로 히스토리를 요약하여 세션 부하를 최소화한다.

## 5. 품질 게이트 (Quality Gate) 통과 기준

모든 프로젝트는 다음의 **'3無 원칙'**을 통과해야 완료로 간주한다.

1.  **무오류 (No Error):** 문서 내 언급된 모든 코드 참조(Path/Symbol)가 실제 존재해야 한다.
2.  **무패닉 (No Panic):** 모든 PoC 코드는 `go run` 시 에러 없이 의도한 결과를 출력해야 한다.
3.  **무누락 (No Omission):** 연구 단계에서 정의된 핵심 서브시스템에 대한 PoC가 최소 1개 이상 존재해야 한다.
