# CNCF EDU 프로젝트 검증 계획 및 현황 (Validation Plan & Status)

이 문서는 클로드(Claude Max)가 생성한 모든 교육 자료(EDU)의 기술적 정확성을 Google AI Pro와 Gemini CLI의 도구를 활용하여 검증하기 위한 마스터 플랜입니다.

## 1. 검증 목표 (Validation Goals)

*   **실행 가능성 보장:** 모든 PoC `main.go` 코드가 추가 의존성 없이 로컬 환경에서 즉시 실행되어야 함.
*   **기술적 일관성:** 문서 내에서 인용된 파일 경로, 함수명, 구조체명이 실제 프로젝트 소스코드와 100% 일치해야 함.
*   **심층 분석의 정확성:** "왜(Why)"에 대한 분석 내용이 실제 소스코드의 설계 의도와 부합해야 함 (Hallucination 방지).

## 2. 검증 항목 (Checklist)

| 항목 | 검증 방법 | 도구 |
| :--- | :--- | :---: |
| **파일 경로 (Path)** | 문서 내 언급된 모든 파일의 존재 여부 확인 | `ls` |
| **코드 심볼 (Symbol)** | 함수, 구조체, 인터페이스 선언문 확인 | `grep` |
| **PoC 실행 (Runtime)** | `go run main.go` 컴파일 및 실행 결과 확인 | `run_shell_command` |
| **내용 정합성 (Audit)** | 1M+ 토큰 컨텍스트를 활용한 로직 교차 검증 | Gemini Pro |

## 4. 향후 검증 실행 계획

1.  **Batch Validation (기존 11개):** 이미 완료된 프로젝트들에 대해 순차적으로 '경로 존재 여부'와 'PoC 실행 여부'를 일괄 검사합니다.
2.  **Istio 집중 검증:** 현재 진행 중인 Istio의 경우, 클로드가 각 문서를 생성할 때마다 Gemini가 실시간으로 소스코드와 대조하여 피드백을 줍니다.
3.  **오류 수정(Fix-loop):** 검증 과정에서 발견된 오류는 Gemini가 직접 파일 내용을 수정하거나, 클로드에게 수정 요청을 보낼 수 있는 가이드를 작성합니다.
