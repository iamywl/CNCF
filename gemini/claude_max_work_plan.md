# Claude Max 최적화 작업 계획서 (CNCF EDU 프로젝트)

이 계획서는 Claude Max(3.5 Sonnet 등)의 **200k 컨텍스트 윈도우**와 **메시지 제한**을 고려하여, 토큰 소모를 최소화하면서 500줄 이상의 고품질 문서를 뽑아내기 위한 전략적 가이드입니다.

## 1. 핵심 토큰 관리 전략

1.  **Context Package (Gemini CLI 역할):** Claude에게 수만 라인의 소스코드를 직접 읽히는 것은 토큰 낭비입니다. Gemini CLI가 먼저 코드를 분석하여 핵심 구조, 주요 함수, 데이터 모델을 요약한 '컨텍스트 패키지'를 Claude에게 전달합니다.
2.  **Batching (분할 요청):** 12개의 심화 문서를 한 번에 요청하지 않습니다. 3~4개씩 묶어서 요청하여 출력 품질 저하와 메시지 제한에 따른 끊김을 방지합니다.
3.  **Session Reset:** 프로젝트 하나가 완료될 때마다 반드시 새 대화를 시작합니다. 누적된 대화 이력은 답변의 정확도를 떨어뜨리고 불필요한 입력 토큰을 소모합니다.
4.  **Incremental Expansion:** "500줄 이상 작성해줘"라고 한 번에 요청하기보다, "목차를 잡고 -> 섹션 1, 2를 상세히 쓰고 -> 나머지를 확장해줘"라는 식으로 단계적으로 요청하여 분량을 확보합니다.

## 2. 프로젝트별 실행 단계 (5단계 프로세스)

### [1단계] Research & Context Setup (Gemini CLI)
*   **작업:** `grep_search`로 진입점 파악, `read_file`로 핵심 데이터 구조 추출.
*   **산출물:** `claude_context_istio.md` 등 프로젝트별 요약 파일 생성.
*   **효과:** Claude의 리서치용 메시지 턴을 10회 -> 1회로 단축.

### [2단계] Architecture & Basic Docs (Claude Max)
*   **입력:** Gemini가 만든 컨텍스트 패키지.
*   **작업:** README.md 및 01~06 기본 문서 작성.
*   **토큰 활용:** 아키텍처 이해에 집중하여 심화 문서의 기초를 다짐.

### [3단계] Deep-Dive Docs Batch 1~3 (Claude Max)
*   **작업:** 심화 문서 12개를 3회에 걸쳐 작성 (회당 4개씩).
*   **프롬프트:** "제공된 소스코드 참조를 바탕으로, 각 문서를 최소 500줄 이상, 코드 구현의 'Why'를 중심으로 상세히 작성해줘."
*   **병목 관리:** 메시지 제한이 걸리면 다음 리셋 시간까지 대기.

### [4단계] PoC Development (Claude Max)
*   **작업:** 16~18개의 PoC `main.go` 작성.
*   **전략:** 실제 소스코드의 알고리즘을 표준 라이브러리만으로 재현하도록 유도.

### [5단계] Validation & Refinement (Gemini CLI)
*   **작업:** Claude가 쓴 PoC 코드를 로컬에서 `go run`으로 실행 검증.
*   **작업:** 문서 내 파일 경로 및 함수명 실제 존재 여부 재확인.
