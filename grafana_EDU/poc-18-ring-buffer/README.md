# PoC 18: 시계열 링 버퍼

## 목적

고정 크기 링 버퍼(Ring Buffer)를 사용하여 시계열 데이터를 효율적으로 저장하는
패턴을 시뮬레이션한다. 버퍼가 가득 차면 가장 오래된 데이터를 덮어쓰며,
시간 범위 쿼리, 집계 함수, 다운샘플링 기능을 구현한다.

## Grafana 관련 개념

Grafana는 시계열 데이터를 다루는 핵심 플랫폼이다.
데이터 소스에서 가져온 시계열을 메모리에 효율적으로 버퍼링하고,
대시보드 쿼리에 빠르게 응답하기 위해 링 버퍼 패턴을 활용한다:

- **Live 스트리밍 데이터**: WebSocket으로 수신되는 실시간 메트릭
- **메모리 관리**: 고정 크기로 메모리 사용량 제한
- **시간 범위 쿼리**: 특정 시간 범위의 데이터만 효율적으로 추출

## 핵심 개념

### 링 버퍼 구조

```
용량: 8, 현재 7개 데이터 저장 (head=0, tail=7)

  인덱스: 0    1    2    3    4    5    6    7
         ┌────┬────┬────┬────┬────┬────┬────┬────┐
  데이터: │ D0 │ D1 │ D2 │ D3 │ D4 │ D5 │ D6 │    │
         └────┴────┴────┴────┴────┴────┴────┴────┘
          ▲                                   ▲
         head                                tail

Push 2개 더 (D7, D8) → D0 덮어쓰기:

  인덱스: 0    1    2    3    4    5    6    7
         ┌────┬────┬────┬────┬────┬────┬────┬────┐
  데이터: │ D8 │ D1 │ D2 │ D3 │ D4 │ D5 │ D6 │ D7 │
         └────┴────┴────┴────┴────┴────┴────┴────┘
               ▲                              ▲
              head                            tail (→ D8에서 wrap)
```

### 시간 범위 쿼리

```
버퍼 내용 (시간순):
  T=10:00  T=10:01  T=10:02  T=10:03  T=10:04  T=10:05

Range(10:01, 10:04):
  → [T=10:01, T=10:02, T=10:03, T=10:04]
```

### 다운샘플링

```
원본 (1분 간격, 60개 포인트):
  10:00, 10:01, 10:02, ..., 10:59

5분 간격으로 다운샘플링 (12개 포인트):
  10:00 (mean of 10:00~10:04)
  10:05 (mean of 10:05~10:09)
  ...
  10:55 (mean of 10:55~10:59)
```

## 실행

```bash
go run main.go
```

## 출력 예시

```
=== 시계열 링 버퍼 시뮬레이션 ===

[Push] 200개 포인트 추가 (용량: 100)
  → 100개 덮어쓰기 발생
  → 현재 버퍼: 100개 (가득 참)
  → 가장 오래된: T=100, 가장 최신: T=199

[Range] T=150 ~ T=170
  → 21개 포인트 반환

[Aggregation] 전체 버퍼
  Mean: 149.5
  Max:  199.0
  Min:  100.0
  Sum:  14950.0
  Count: 100
```

## 학습 포인트

1. **고정 크기 메모리**: 링 버퍼로 메모리 사용량을 예측 가능하게 관리
2. **O(1) 삽입**: 인덱스 wrap-around로 할당 없는 빠른 삽입
3. **시간 기반 쿼리**: 이진 탐색으로 효율적인 시간 범위 검색
4. **다운샘플링**: 오래된 데이터의 해상도를 낮춰 저장 공간 절약
5. **다중 시리즈**: 레이블 기반 시리즈 관리로 여러 메트릭 동시 처리
