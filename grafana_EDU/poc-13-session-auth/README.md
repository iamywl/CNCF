# PoC 13: 세션 인증

## 목적

Grafana의 토큰 기반 세션 인증 시스템을 시뮬레이션한다.
사용자 로그인 시 세션 토큰을 발급하고, 요청마다 토큰을 검증하며,
보안을 위해 주기적으로 토큰을 회전(rotate)하는 전체 수명주기를 구현한다.

## Grafana 실제 구현 참조

Grafana는 `pkg/services/auth/` 패키지에서 세션 인증을 처리한다:

- **UserToken**: 토큰 해시, 사용자 ID, 생성/회전/만료 시각을 포함하는 핵심 구조체
- **TokenService**: 토큰 CRUD 및 회전 로직 담당
- **grafana_session 쿠키**: 브라우저에 토큰을 전달하는 HTTP 쿠키

## 핵심 개념

### 토큰 수명주기

```
로그인 요청
    │
    ▼
CreateToken (랜덤 32바이트 → hex → SHA256 해시 저장)
    │
    ▼
Set-Cookie: grafana_session=<hex_token>
    │
    ▼
┌─────────────────────────────┐
│   요청마다 LookupToken      │◄──── Cookie에서 토큰 추출
│   ├─ 만료 확인              │
│   ├─ 비활성 시간 확인        │
│   └─ 회전 필요 여부 확인     │
└──────────┬──────────────────┘
           │
     회전 필요?
    ┌──┴──┐
   Yes    No
    │      │
    ▼      ▼
RotateToken  요청 처리
(새 토큰 발급,
 유예 기간 동안
 이전 토큰 유효)
```

### 보안 설계

| 설정 | 값 | 설명 |
|------|-----|------|
| 토큰 형식 | 32바이트 랜덤 hex | 추측 불가능한 토큰 |
| 저장 방식 | SHA256 해시 | DB 유출 시에도 안전 |
| 회전 주기 | 10분 | 탈취 토큰의 유효 시간 최소화 |
| 유예 기간 | 30초 | 동시 요청 시 이전 토큰 허용 |
| 최대 수명 | 30일 | 토큰 절대 만료 시간 |
| 비활성 만료 | 7일 | 미사용 세션 자동 정리 |
| 동시 세션 | 3개 | 사용자당 최대 활성 세션 수 |

## 실행

```bash
go run main.go
```

## 출력 예시

```
=== Grafana 세션 인증 시뮬레이션 ===

[LOGIN] user=admin (ID=1)
  → 토큰 생성: a3f2c1...
  → 쿠키 설정: grafana_session=a3f2c1...
  → 만료: 2026-04-04, 비활성 만료: 2026-03-12

[REQUEST] GET /api/dashboards
  → 쿠키에서 토큰 추출: a3f2c1...
  → LookupToken: 유효 (남은 수명: 29일)
  → 회전 불필요 (마지막 회전: 3분 전)

[REQUEST] GET /api/datasources (12분 후)
  → 토큰 회전 필요 (마지막 회전: 12분 전 > 10분)
  → 새 토큰: b7e9d4...
  → 이전 토큰 유예 기간: 30초
  → 쿠키 갱신: grafana_session=b7e9d4...

[REVOKE] 로그아웃 user=admin
  → 토큰 폐기: b7e9d4...
```

## 학습 포인트

1. **토큰 해싱**: 원본 토큰 대신 SHA256 해시를 저장하여 DB 유출 시에도 세션 탈취 불가
2. **토큰 회전**: 주기적 회전으로 탈취된 토큰의 유효 기간 최소화
3. **유예 기간**: 회전 시 이전 토큰에 유예 기간을 두어 동시 요청 처리
4. **동시 세션 제한**: 사용자당 최대 활성 세션 수로 무분별한 세션 생성 방지
5. **이중 만료**: 절대 만료(30일)와 비활성 만료(7일)의 조합
